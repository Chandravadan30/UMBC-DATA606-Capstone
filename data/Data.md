# Datasets Used in the Project

This project uses three real public datasets to model user behavior, operational burden, and technical risk for the fictional e-commerce application **Lunara**.

---

## 1️⃣ Google Analytics Sample Dataset

**Official Link:**  
https://developers.google.com/analytics/bigquery/web-ecommerce-demo-dataset  

**Description:**  
This dataset contains real user session data from an e-commerce website.  
Each row represents one user session including page views, transactions, device type, and traffic source.  
It is used to model user behavior, feature adoption trends, and engagement levels over time.

---

## 2️⃣ GitHub Public Issues Dataset

**Official Link:**  
https://cloud.google.com/bigquery/public-data/github  

**Description:**  
This dataset contains real issue reports from open-source software repositories.  
Each row represents one issue such as a bug report, enhancement request, or complaint.  
It is used as a proxy for operational burden and user-reported problems affecting product features.

---

## 3️⃣ NASA Software Defect Dataset

**Official Link:**  
https://www.kaggle.com/datasets/semustafacevik/software-defect-prediction  

**Description:**  
This dataset contains software quality and defect metrics from NASA systems.  
Each row represents one software module with complexity measurements and defect indicators.  
It is used as a proxy for technical debt and maintenance risk in feature lifecycle decisions.
